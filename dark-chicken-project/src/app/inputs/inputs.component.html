Darkchicken Inputs



This page is designed to introduce the versatile and powerful dca-input components available in the DarkChicken library for Angular developers. Our input components are created to enhance user experience and expedite the development process with extensive customization options and built-in validation features.

Key Features

  • Customizability: Each input component can be tailored to meet your needs, with options to set labels, placeholders, read-only states, visibility, and more.
  • Validation: Our input components support comprehensive validation rules, including required fields, minimum and maximum values, length restrictions, and regular expressions. Validation messages provide immediate feedback to users.
  • Event Handling: dca-input components emit various events such as value changes, focus in and out, validation states, and mouse interactions. These events enable developers to create responsive and interactive forms.
  • Icon Integration: Enhance your input fields with integrated icons to provide visual cues and improve user interface aesthetics.

Usage Examples

Default Input Fields

  • Simple input fields with labels and placeholders.
  • Read-only and disabled states to control user interaction.
  • Visibility control to dynamically show or hide input fields.

Outlined Input Fields

  • Input fields with validation rules for required fields, minimum and maximum values, length restrictions, and regular expressions.
  • Examples demonstrating how to provide feedback for various validation scenarios.

Interactive Input Fields

  • Input fields with event bindings to handle value changes, focus events, and mouse events.
  • Integrated icons to visually enhance input fields and provide contextual information.
